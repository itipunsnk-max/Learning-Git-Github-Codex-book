Branch, merge และ conflict

Branch คือเส้นทางทำงานแยกจาก main ช่วยให้พัฒนา feature หรือแก้ bug โดยไม่รบกวนงานที่พร้อมใช้งาน

Workflow ที่แนะนำ

git switch main
git pull --rebase
git switch -c feature/customer-search
# แก้ไฟล์ ตรวจ diff และ commit
git push -u origin feature/customer-search
# เปิด Pull Request เพื่อ review
git switch main
git pull --rebase
git merge feature/customer-search
git branch -d feature/customer-search

ชื่อ branch ควรบอกเจตนา เช่น feature/customer-search, fix/date-parsing, docs/git-guide

Fetch, pull และ remote

git remote -v
git fetch origin
git branch -a
git pull --rebase
คำสั่งผลลัพธ์
fetchดาวน์โหลดข้อมูล remote แต่ไม่รวมเข้ากับไฟล์งาน
pullfetch แล้ว merge หรือ rebase ตามที่กำหนด
pushส่ง commit จาก local ไป remote

แก้ Merge Conflict

เมื่อสอง branch แก้บรรทัดเดียวกัน Git จะใส่ marker ให้เปิดดู:

<<<<<<< HEAD
ข้อความจาก branch ปัจจุบัน
=======
ข้อความจาก branch ที่นำมารวม
>>>>>>> feature/example
  1. อ่านทั้ง Current และ Incoming change
  2. เลือกหรือเขียนผลลัพธ์ที่ถูกต้อง แล้วลบ marker ให้หมด
  3. บันทึกและทดสอบโปรแกรม
  4. ใช้ git add <resolved-file> แล้ว git commit
git merge --abort
git rebase --abort
อย่ากด Accept Both โดยไม่อ่าน: ผลลัพธ์อาจรันได้แต่ผิด logic
