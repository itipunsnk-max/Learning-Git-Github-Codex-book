Codex กับ repository

ให้ Codex ช่วยแก้ปัญหาได้เร็วขึ้นด้วยขอบเขตที่ชัด ตรวจ diff ทุกครั้ง และให้ผลลัพธ์ที่ทดสอบซ้ำได้

ก่อนเริ่มงาน

  1. ตรวจ branch และสถานะ: git status
  2. บันทึกหรือ stash งานค้างที่ไม่เกี่ยวข้อง
  3. ระบุไฟล์/โฟลเดอร์ที่ Codex แก้ได้และแก้ไม่ได้
  4. บอก acceptance criteria, test command และข้อจำกัดด้าน secret
git switch -c fix/codex-task
git status

หลัง Codex แก้เสร็จ

git status
git diff --stat
git diff
git diff --check
# รัน test / lint
git diff --staged

ถ้ามีไฟล์เกินขอบเขต ให้หยุดและตรวจเหตุผลก่อน stage อย่า commit ทุกอย่างด้วย git add . โดยไม่อ่าน diff

ความปลอดภัย: อย่าใส่ token, password, private key, customer data หรือข้อมูลส่วนบุคคลใน prompt, example หรือ commit

โครง prompt ที่ใช้ซ้ำได้

เป้าหมาย: แก้ปัญหา ...
ขอบเขต: แก้เฉพาะ ...
ห้ามแก้: ...
เกณฑ์สำเร็จ: ...
การตรวจสอบ: รัน ... แล้วสรุปผล
ก่อนจบ: แสดงไฟล์ที่เปลี่ยนและความเสี่ยงที่เหลือ

Codex ช่วยเสนอและลงมือแก้ได้ แต่ผู้ดูแล repository ยังต้องเป็นผู้ตัดสินใจเรื่อง scope, correctness, security และการ commit
