02 · Inspect

Diff และการ review งาน

ก่อน stage หรือ commit ให้ดูสิ่งที่เปลี่ยนจริงเสมอ เพราะสถานะไฟล์บอกว่ามีงานค้าง แต่ diff บอกว่าคืองานอะไร

คำสั่งที่ใช้บ่อย

git status
git status --short
git diff
git diff --staged
git diff HEAD
git diff --stat
git diff --name-status
git diff --check
คำสั่งใช้เมื่อ
git diffตรวจงานที่แก้แต่ยังไม่ stage
git diff --stagedตรวจ snapshot ที่กำลังจะ commit
git diff main..feature/nameเปรียบเทียบสอง branch
git diff --checkหา whitespace error ก่อน commit

Review loop แบบมืออาชีพ

  1. อ่าน git status และยืนยัน branch ปัจจุบัน
  2. ใช้ git diff เช็กว่าการแก้ยังอยู่ในขอบเขตงาน
  3. stage เฉพาะไฟล์/ส่วนที่พร้อมส่ง
  4. ใช้ git diff --staged อ่านเหมือน reviewer
  5. รัน test หรือ lint ที่เหมาะสม แล้วค่อย commit
ระวัง: git add . อาจ stage ไฟล์ที่ไม่เกี่ยวข้อง เช่น log, build output หรือ secret

Commit ที่สื่อความหมาย

git commit -m "feat: add customer search"
git log --oneline --graph --decorate --all
git show HEAD

ใช้ Conventional Commits เช่น feat:, fix:, docs:, refactor:, test: และ chore: หนึ่ง commit ควรมีเป้าหมายเดียว
