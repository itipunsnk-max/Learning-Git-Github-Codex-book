01 · Foundation

Git คืออะไร

Git คือระบบ version control ที่เก็บประวัติการเปลี่ยนแปลงไว้ในเครื่องเรา ส่วน GitHub คือบริการ remote สำหรับเก็บ แชร์ และ review repository

ภาพจำ 4 จุด

Working Directory → git add → Staging Area → git commit → Local Repository → git push → GitHub
คำศัพท์ความหมาย
Repositoryโฟลเดอร์โปรเจกต์ที่ Git ติดตามประวัติ
HEADจุดอ้างอิงของ commit ปัจจุบัน
originชื่อ remote เริ่มต้นที่ชี้ไปยัง GitHub
upstreamremote หลักที่ branch ใช้ติดตาม

ติดตั้งและตั้งค่า

เปิด Git Bash, PowerShell หรือ VS Code Integrated Terminal แล้วตรวจว่า Git อยู่ใน PATH

git --version
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
git config --global --list
หมายเหตุ: ใช้ email ที่ตั้งใจให้ปรากฏในประวัติ commit และอย่าใส่ secret ใน config หรือไฟล์โปรเจกต์

สร้าง repository แรก

mkdir my-git-project
cd my-git-project
git init
git status
git add README.md
git diff --staged
git commit -m "docs: add initial README"

git add เลือก snapshot ที่จะ commit ส่วน git commit บันทึก snapshot นั้นใน local history

อ่านสถานะไฟล์

สถานะแปลว่า
Untrackedไฟล์ใหม่ที่ Git ยังไม่ติดตาม
Modifiedไฟล์ tracked ถูกแก้ แต่ยังไม่ stage
Stagedเลือกไว้ใน snapshot ถัดไป
Committed / Cleanบันทึกแล้วและไม่มีงานค้าง
ConflictedGit รวมการเปลี่ยนแปลงให้เองไม่ได้ ต้องตัดสินใจ
