07 · Field notes

Troubleshooting

แก้ปัญหาโดยเก็บหลักฐานทีละชั้น: เราอยู่ที่ไหน, Git เห็นอะไร, remote คืออะไร และคำสั่งล่าสุดทำให้สถานะเปลี่ยนอย่างไร

ลำดับวิเคราะห์มาตรฐาน

Get-Location          # PowerShell
git rev-parse --show-toplevel
git status
git branch --show-current
git remote -v
git log --oneline -5

อาการที่พบบ่อย

อาการตรวจ/แนวทาง
git is not recognizedติดตั้ง Git ใหม่ เปิด terminal ใหม่ และตรวจ PATH ด้วย git --version
Source Control ว่างเปิด folder ที่มี .git และตรวจ current directory
remote origin already existsดู git remote -v แล้วใช้ git remote set-url origin <url>
repository not foundตรวจ URL, สิทธิ์ repository และบัญชีที่ credential manager ใช้
non-fast-forwardใช้ git fetch ตรวจความต่าง แล้ว git pull --rebase หากเหมาะสม
authentication failedGitHub ไม่รับ password แบบเดิม ให้ใช้ credential manager, SSH หรือ token ตามนโยบายองค์กร

ก่อนขอความช่วยเหลือ

  • คัดลอก error message โดยตัด token และข้อมูลส่วนตัวออก
  • บอก OS, shell, branch และคำสั่งที่รัน
  • แสดง git status และ git remote -v เฉพาะ URL ที่เปิดเผยได้
  • อย่าลบหรือ reset เพื่อ “ลองดู” หากยังไม่ได้สำรอง/สร้าง recovery branch
