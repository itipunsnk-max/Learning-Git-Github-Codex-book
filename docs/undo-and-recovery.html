04 · Recover

ย้อนและกู้คืนงาน

อ่านสถานะก่อนย้อนทุกครั้ง คำสั่งบางอย่างลบการเปลี่ยนแปลงที่ยังไม่มี commit ได้ถาวร

เลือกคำสั่งจากจุดที่งานอยู่

สถานการณ์คำสั่งเริ่มต้น
แก้ไฟล์แล้ว ยังไม่ addgit restore <file>
add แล้ว ยังไม่ commitgit restore --staged <file>
commit แล้ว ยังไม่ pushgit reset --soft HEAD~1 หรือ mixed
push แล้ว / sharedgit revert <commit>
หา commit ที่หายgit reflog แล้วสร้าง recovery branch
หยุดก่อนใช้ git reset --hard: ตรวจ git status, git diff และ commit เป้าหมายก่อนเสมอ

Reset กับ Revert

git reset --soft HEAD~1
git reset --mixed HEAD~1
git reset --hard HEAD~1
git revert HEAD
resetrevert
เปลี่ยนประวัติ เหมาะกับ local commitสร้าง commit ใหม่ เหมาะกับ shared commit
อาจต้อง force pushpush ตามปกติและปลอดภัยกว่าสำหรับทีม

Reflog: กู้ commit ที่หาย

git reflog
git switch -c recovery/before-reset <commit-hash>
# ตรวจงานบน branch ก่อน แล้วค่อยตัดสินใจ

การสร้าง recovery branch ก่อน reset เป็นทางเลือกที่ปลอดภัยกว่า เพราะยังมีชื่ออ้างอิงให้ย้อนกลับ

Stash งานชั่วคราว

git stash push -m "wip: customer report"
git stash list
git stash show -p stash@{0}
git stash apply
git stash pop
git stash -u

apply นำงานกลับแต่เก็บ stash ไว้ ส่วน pop นำกลับแล้วลบรายการเมื่อสำเร็จ อย่าใช้ git stash clear หากยังไม่แน่ใจว่างานถูกเก็บที่อื่นแล้ว
