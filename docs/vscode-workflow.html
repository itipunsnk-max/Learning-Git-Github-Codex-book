05 · Editor

Git workflow ใน VS Code

VS Code เป็นหน้าต่างทำงานกับ repository: แก้ไฟล์, ดู diff, stage, commit และเปิด terminal ได้ในบริบทเดียวกัน

เปิด folder ให้ถูก

  1. ใช้ File → Open Folder แล้วเลือกโฟลเดอร์ที่มี .git
  2. เปิด Terminal ใหม่และตรวจ pwd หรือ Get-Location
  3. รัน git status เพื่อยืนยัน repository และ branch
  4. ถ้า Source Control ไม่แสดง ให้ตรวจว่าเปิดโฟลเดอร์ชั้นนอก/ชั้นในถูกหรือไม่

Source Control แบบตรวจสอบได้

รายการ Changes คือไฟล์ที่ยังไม่ stage; Staged Changes คือ snapshot ที่พร้อม commit คลิกไฟล์เพื่อเปิด diff editor แล้วอ่านบรรทัดเพิ่ม/ลบก่อนกด Stage

การกระทำคำแนะนำ
Stageเลือกไฟล์หรือบางส่วนของไฟล์ที่เกี่ยวข้อง
Commitใช้ข้อความที่บอก intent เช่น fix: handle empty result
Discardใช้หลังตรวจ diff แล้วเท่านั้น เพราะลบงานที่ยังไม่ commit
Resolve conflictอ่าน Current/Incoming, บันทึก, stage และทดสอบ

Terminal และ keyboard-first

git status
git diff
git diff --staged
git log --oneline --graph --decorate --all
หลักปฏิบัติ: ใช้ UI เพื่อเห็นภาพและใช้ terminal เพื่อคำสั่งที่ต้องการความแม่นยำ เกณฑ์ตัดสินใจควรเหมือนกันทั้งสองทาง
